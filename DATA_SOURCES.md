# Data Sources

## Context

This project aims to pre-train on a broad AMP corpus assembled from multiple databases, not just a small UniProt-only subset.

The canonical training corpus is generated by:

```bash
uv run python scripts/prepare_amp_dataset.py
```

and written to:

- `data/processed/amp_corpus.fasta`
- `data/processed/amp_source_stats.json`
- `data/processed/amp_sequence_manifest.tsv`

## Execution

### Auto-download sources

- `UniProt reviewed AMP`
  - file: `data/raw/uniprot_amps.fasta`
  - current script support: yes
- `UniProt TrEMBL AMP`
  - file: `data/raw/uniprot_amps_trembl.fasta`
  - current script support: yes
- `APD6 natural AMP`
  - file: `data/raw/apd3.fasta`
  - current script support: yes
  - official page: `https://aps.unmc.edu/downloads`
- `DRAMP natural AMP`
  - file: `data/raw/dramp_amps.fasta`
  - current script support: yes, but may fail due DNS / site availability

### Semi-automatic or manual sources

- `DBAASP`
  - file: `data/raw/dbaasp_amps.fasta`
  - public mirrors may disappear
  - if a direct export is not accessible, place a FASTA export here manually, or drop one or more FASTA files into `data/raw/manual/`
- `AMPlify / benchmark mirrors`
  - file: `data/raw/amplify_amps.fasta`
  - optional supplemental source
- `Additional manual exports`
  - directory: `data/raw/manual/*.fasta`
  - any FASTA placed here is automatically incorporated by `prepare_amp_dataset.py`

## Insight

- The correct question is not “how many entries exist in theory?” but “how many valid, deduplicated, short AMP sequences do we actually have after filtering?”
- We should track both:
  - database-native counts
  - usable counts after filtering to standard amino acids and target length range
- `data/processed/amp_source_stats.json` is the authoritative local answer for the current workspace.
- DBAASP web export currently requires reCAPTCHA and segmented downloads, so the practical workflow is often: browser export -> save FASTA -> place in `data/raw/manual/` -> rerun `prepare_amp_dataset.py`.

## Current Goal

- Short term: push the processed corpus beyond the old `1627`-sequence UniProt-only state.
- Medium term: reach a stable multi-source corpus large enough to justify JEPA pre-training.
- Long term: separate natural-only and mixed-source corpora so pre-training assumptions are explicit.
